# CineVault

CineVault — локальное Windows-приложение для ведения личной библиотеки
фильмов и выбора следующего фильма через колесо с механикой выбывания.

## Статус

Версия `0.9.0-beta` содержит полный основной пользовательский цикл:
каталог, категории и очереди, франшизы, колесо с сейвами, просмотренные
фильмы, оценки, статистику, историю сессий, резервные копии и импорт
CSV/TSV/XLSX.

Старый код сохранён в каталоге `legacy/` только как исторический референс.
Новая реализация находится в `src/`.

## Основные возможности

- фильмы, категории, подкатегории и франшизы;
- ручная очередь фильмов внутри категории;
- квота участников колеса для каждой категории;
- колесо в формате батл-рояля;
- персональные сейвы участников;
- просмотренные фильмы и оценки с шагом 0,5;
- история завершённых сессий;
- импорт существующей библиотеки и резервное копирование;
- полностью русскоязычный интерфейс в золотом стиле CineVault.

## Локальный запуск

На Windows при установленном Python 3:

```powershell
python launch.py
```

Лаунчер подберёт свободный локальный порт и откроет приложение в браузере.
Прямое открытие `index.html` не рекомендуется, поскольку приложение
использует ES-модули и IndexedDB.

Также приложение можно запустить двойным кликом по `CineVault.cmd`.

## Переносимая сборка

```powershell
powershell -ExecutionPolicy Bypass -File .\build-portable.ps1
```

Скрипт создаст:

- `release/CineVault/` — чистую папку приложения;
- `release/CineVault-portable.zip` — архив для переноса.

На целевом компьютере требуется Python 3. Установка зависимостей не нужна.

## Проверка

Модульные тесты используют встроенный test runner Node.js:

```powershell
npm test
python -m unittest discover -s tests -p "test_*.py"
```

## Принцип разработки

Новые функции не добавляются непосредственно в `legacy/`. Сначала
фиксируются модель данных и бизнес-правила, затем создаётся новая версия
приложения с тестируемыми модулями.
