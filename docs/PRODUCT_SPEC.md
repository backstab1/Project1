# CineVault — продуктовая спецификация v1

## Назначение

CineVault — личное локальное приложение для Windows, которым обычно
пользуются два зрителя. Оно хранит библиотеку полнометражных фильмов,
организует её по пользовательским категориям и выбирает следующий фильм
через колесо с последовательным выбыванием.

## Ограничения первой версии

- до 500 фильмов;
- только русский интерфейс;
- минимальное разрешение 1366×768;
- обязательная офлайн-работа;
- повторные просмотры не учитываются;
- авторизация и облачная учётная запись отсутствуют;
- автоматические метаданные являются необязательным онлайн-дополнением.

## Данные

### Фильм

Обязательные поля:

- уникальный идентификатор;
- локализованное название.

Дополнительные поля:

- оригинальное название;
- категория;
- позиция в очереди категории;
- URL постера;
- год;
- продолжительность;
- страна;
- дата просмотра;
- оценки зрителей.

Фильм принадлежит максимум одной категории и максимум одной франшизе.

### Категория

Категория является пользовательской группировкой, а не обязательно жанром.
Она может иметь родительскую категорию, ручную позицию и квоту элементов
для формирования колеса.

Удаление категории не удаляет фильмы. Они переходят в состояние
«Без категории».

### Франшиза

Франшиза — самостоятельный участник каталога и колеса. Она содержит
упорядоченный список фильмов и не может включать другие франшизы.

Статус просмотра франшизы не хранится отдельно: он вычисляется по её
фильмам. Франшиза просмотрена, когда просмотрены все входящие фильмы.

### Оценка

- значение от 1 до 10;
- шаг 0,5;
- одно значение на имя зрителя;
- повторный ввод заменяет предыдущее значение;
- оценку разрешено изменить или удалить.

Фильмы без оценок не участвуют в вычислении рейтинга франшизы.

### Сессия колеса

При запуске создаётся снимок участников. Изменение каталога не меняет
активную сессию.

Сессия хранит:

- исходный пул;
- текущий пул;
- выбывших;
- участников и остатки сейвов;
- порог отключения сейвов;
- журнал действий;
- победителя;
- дату начала и завершения.

Незавершённая сессия не восстанавливается после закрытия приложения.
Завершённые сессии сохраняются в истории.

## Формирование пула

Для каждой категории задаётся квота. В пул попадают первые непросмотренные
элементы согласно ручной очереди категории. Родительская категория может
учитывать элементы дочерних категорий.

Текущая версия правила:

- пул фиксируется перед стартом;
- выбывший участник не заменяется новым во время сессии;
- следующая позиция очереди участвует уже в следующей сессии;
- кнопка «Перемешать» меняет только визуальный порядок элементов.

## Сейвы

Перед стартом задаются участники и количество сейвов каждого. После
выпадения элемента игрок может потратить сейв, чтобы отменить выбывание.

- сейвы действуют только внутри текущей сессии;
- один элемент можно спасать несколько раз;
- сейвы нельзя передавать или добавлять после старта;
- пользователь задаёт число оставшихся элементов, после которого сейвы
  блокируются;
- применение сейва записывается в журнал.

## Победитель

Если выигрывает фильм, он получает текущую дату просмотра. Если выигрывает
франшиза, одну дату просмотра получают все её фильмы. После этого
предлагается поставить оценки, но окно можно закрыть без оценки.

## Хранение и перенос

Основное хранилище — IndexedDB с версией схемы. В дальнейшем добавляются:

- миграция старых ключей `localStorage`;
- версионный экспорт JSON;
- объединяющий импорт;
- импорт согласованного CSV/XLSX;
- обнаружение дубликатов;
- напоминание о резервном копировании.
