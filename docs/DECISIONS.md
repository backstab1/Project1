# Решения и открытые вопросы

## Зафиксированные решения

- приложение личное, обычно используется двумя зрителями;
- платформа — Windows, минимальное разрешение 1366×768;
- интерфейс только на русском языке;
- обязательная работа без интернета;
- объём библиотеки — до 500 фильмов;
- архитектуру исходного прототипа разрешено полностью заменить;
- фильм находится только в одной категории;
- категории поддерживают иерархию;
- порядок категорий и фильмов задаётся вручную;
- удаление категории переносит фильмы в «Без категории»;
- фильм входит максимум в одну франшизу;
- франшиза не содержит другие франшизы;
- победа франшизы отмечает просмотренными все входящие фильмы;
- повторные просмотры не хранятся;
- оценка имеет шаг 0,5;
- один зритель имеет одну оценку на фильм;
- новая оценка зрителя заменяет предыдущую;
- незавершённая сессия не сохраняется;
- завершённые сессии сохраняются;
- шансы всех участников равны;
- сейвы принадлежат игрокам и расходуются только внутри сессии;
- приложение использует золотой стиль CineVault;
- пробел запускает вращение колеса;
- основной слой хранения новой версии — IndexedDB.

## Рабочие решения до отдельного согласования

Эти решения приняты для продолжения разработки и могут быть изменены:

1. Пул фиксируется при старте сессии.
2. Выбывший участник не заменяется следующим элементом очереди во время
   текущего батл-рояля.
3. Очередь сдвигается при формировании следующей сессии.
4. До подтверждения выбывания разрешена перекрутка без лимита.
5. Удаление франшизы сохраняет её фильмы как самостоятельные.
6. Оценку победителю можно не ставить сразу.
7. Вложенность категорий не ограничивается моделью данных.
8. Автоматическая загрузка метаданных является необязательной онлайн-функцией
   и не мешает ручной офлайн-работе.
9. Первая версия синхронизации — экспорт и импорт резервного файла.
10. Франшиза занимает одну позицию в категории и очереди.

## Требуется от заказчика

- пример существующей Google Таблицы или экспорт в XLSX/CSV;
- окончательное подтверждение поведения автоматической подстановки
  следующего фильма из очереди;
- предпочтительный источник автоматических метаданных.
